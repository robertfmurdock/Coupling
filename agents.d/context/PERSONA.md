# Persona

The repository owner is named RoB Murdock.

## About Me
- Appreciates a little whimsy.
- I've been working in software since 2005, in many, many languages.
- I tend to focus on big picture ideas, and ensuring that real value is the "north star."

## Working Values
- Clarity is to be preferred over optimization
- Code must be run to know what it does 
- tests must fail for their intended reason to have meaning - tests that have never failed prove nothing
- if it hurts, do it more... then optimize it!
- show your work
- keep functions small, files small.
- name well.
- duplication is to be avoided, but is sometimes appropriate given the expected evolution of a system
- preoptimization is to be avoided.  
- prefer functional styles over class extention.

## Collaboration Preferences
- Start by confirming assumptions that affect architecture or scope.
- Surface risks early, with concrete options and impact.
- Keep updates concise and action-oriented.
- When blocked, propose the next best path rather than stalling.

## Decision Heuristics
- Prefer established project conventions unless there is a measurable gain
- Minimize blast radius for each change.
- Match test depth to risk and surface area.
- Choose boring, dependable solutions for critical paths.

## Anti-Goals
- Unwieldy build logic.
- New abstractions without clear payoff.

## Done Criteria
- Tests describe all new features, and have failed for correct reasons before implementation was done. 
- Behavior aligns with intentions
- Changes are scoped, readable, and easy to review.
- Any concerns that the changes have raised are escalated accordingly
